Composable table controls for Nuxt UI applications. `@querry-kit/nuxt-ui` provides collision-safe `QuerryKit*` components for table toolbars, sorting, filtering, column preferences and pagination. It keeps table data, query state and application stores in the consuming application.

📖 Documentation: https://querry-kit.github.io/nuxt-ui/

Explore the wider [@querry-kit/querry-kit](https://querry-kit.github.io/querry-kit/) overview for the three core repositories—Nest, Nuxt, and Nuxt UI—and its comprehensive API and web application example.

## 📦 Install

```sh
pnpm add @querry-kit/nuxt @querry-kit/nuxt-ui @nuxt/ui
```

`@nuxtjs/i18n` is optional. Install it when the module should merge the included English and German table messages:

```sh
pnpm add @nuxtjs/i18n
```

The current package version is published on npm. npm is the primary distribution channel.

## 🚀 Release Workflow

Releases are driven by Changesets and GitHub Actions. Package-visible changes should include a changeset:

```sh
pnpm changeset
```

When changes land on `main`, the `changesets` workflow creates or updates a release PR. That PR contains the version bump and changelog generated by:

```sh
pnpm changeset version
```

After the release PR is merged, the `npm publish` workflow runs linting, type checks, tests with coverage, the package build and the documentation build. It then publishes the public package with npm Trusted Publishing, tags the release as `vX.Y.Z`, and creates a GitHub Release.

Consumers should install from npm:

```sh
pnpm add @querry-kit/nuxt-ui
```

## 🧩 Usage

Register Nuxt UI before this module. Components are auto-imported, and the optional locale integration merges only `querryKit.table.*` messages.

```ts
export default defineNuxtConfig({
  modules: ['@nuxt/ui', '@querry-kit/nuxt-ui'],
  querryKitUi: {
    autoImports: true,
    locales: true,
  },
});
```

All table state stays in the consuming application. Bind the models needed by the controls you render:

```vue
<QuerryKitTableToolbar
  v-model:search="search"
  v-model:sorting="sorting"
  v-model:filtering="filtering"
  v-model:column-order="columnOrder"
  v-model:invisible-columns="invisibleColumns"
  v-model:column-pinning="columnPinning"
  :sortable-fields="sortableFields"
  :filter-fields="filterFields"
  :column-definitions="columns"
/>
```

The controls emit immutable replacement values. They can therefore be connected to any compatible table, query or store without a dependency on CRM code, application layouts or runtime configuration.

## 🧱 Components

Every public component begins with `QuerryKit` to avoid collisions with application components:

- [`QuerryKitTableToolbar`](https://querry-kit.github.io/nuxt-ui/components/toolbar) composes a breadcrumb, search input, create action and table controls.
- [`QuerryKitTableSorting`](https://querry-kit.github.io/nuxt-ui/components/sorting) manages an ordered sorting state.
- [`QuerryKitTableFiltering`](https://querry-kit.github.io/nuxt-ui/components/filtering) edits Query Kit-compatible filtering values.
- [`QuerryKitTableOptions`](https://querry-kit.github.io/nuxt-ui/components/options) manages column order, visibility and pinning.
- [`QuerryKitTablePagination`](https://querry-kit.github.io/nuxt-ui/components/pagination) renders a standalone page-based API footer.

## 🌍 Locales

Set `locales: false` when the application registers its translation integration itself. The module never installs an i18n provider; it only merges messages when `@nuxtjs/i18n` is already present.

For manual integration, import the locale payloads without installing the module plugin:

```ts
import { querryKitLocales } from '@querry-kit/nuxt-ui/locales';

i18n.global.mergeLocaleMessage('en', querryKitLocales.en);
i18n.global.mergeLocaleMessage('de', querryKitLocales.de);
```

Messages remain namespaced under `querryKit.table`, so application messages stay untouched.
